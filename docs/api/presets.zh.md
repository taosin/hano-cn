# 预设

Hono 有多个路由器，每个路由器都为特定目的设计。
你可以在 Hono 的构造函数中指定想要使用的路由器。

**预设** 为常见用例提供，因此你不必每次都指定路由器。
从所有预设导入的 `Hono` 类是相同的，唯一区别是路由器。
因此，你可以互换使用它们。

## `hono`

用法：

```ts twoslash
import { Hono } from 'hono'
```

路由器：

```ts
this.router = new SmartRouter({
  routers: [new RegExpRouter(), new TrieRouter()],
})
```

## `hono/quick`

用法：

```ts twoslash
import { Hono } from 'hono/quick'
```

路由器：

```ts
this.router = new SmartRouter({
  routers: [new LinearRouter(), new TrieRouter()],
})
```

## `hono/tiny`

用法：

```ts twoslash
import { Hono } from 'hono/tiny'
```

路由器：

```ts
this.router = new PatternRouter()
```

## 我应该使用哪个预设？

| 预设         | 适合的平台                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `hono`       | 这在大多数用例中强烈推荐。虽然注册阶段可能比 `hono/quick` 慢，但一旦启动就表现出高性能。它非常适合使用 **Deno**、**Bun** 或 **Node.js** 构建的长寿命服务器。它也适合 **Fastly Compute**，因为在该平台上路由注册发生在应用程序构建阶段。对于 **Cloudflare Workers**、**Deno Deploy** 等使用 v8 isolates 的环境，这个预设也很合适。因为 isolations 在启动后会持续一段时间。 |
| `hono/quick` | 这个预设为每次请求都初始化应用程序的环境设计。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `hono/tiny`  | 这是最小的路由器包，适合资源有限的环境。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
